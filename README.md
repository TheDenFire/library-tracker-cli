# Library Tracker CLI

Консольное приложение для управления книгами небольшой библиотеки. Приложение позволяет добавлять, удалять, искать и отображать список всех книг с возможностью сортировки.

## Запуск и взаимодействие

Требования:
- Java 21+
- Gradlew из проекта

Команда запуска:

```bash
./gradlew run
```

Помимо этого, есть возможность запуска непосредственно с помощью Idea через основной класс Main.java

После старта приложение принимает команды:

```cmd
ADD <название>;<автор>;<год>
```
```cmd
REMOVE <id>
```
```cmd
LIST [title|author|year]
```
```cmd
FIND <запрос>
```
```cmd
STATS
```
```cmd
EXIT
```

`FIND` принимает один поисковый запрос. Все слова из запроса ищутся в общем тексте `название + автор`, регистронезависимо и по частичному совпадению. Например, `FIND code martin` найдёт `Clean Code / Robert Martin`.

## Принятые решения

Приложение разделено на различные слои, которые разделяют ответственность приложения. Слой `presentation` занимается исключительно вводом/выводом информации и внешним отображением командной строки во время работы приложения.
Слой `repository` реализует inMemory репозиторий на время работы приложения и занимается хранением данных.
Слой `service` связывает между собой presentation и repository, запрашивает у мапперов dto для передачи в тот или иной слой и является "коммутатором" во всем приложении.

Основная и единственная доменная сущность - Book, которая включает в себя такие данные как `id`, `title`, `author` и `year`. Для использовании качественного синглтона `id`, был выбран готовый пакет `UUID`, который внутри себя уже имеет реализованные паттерны уникальности и единственности.

Для взаимодействия между слоями созданы `Data Transfer Object's (DTO)`, которые позволяют иметь контракт взаимодействия и не отдавать доменную сущность на `presentation` слой.
